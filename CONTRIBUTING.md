# Contributing to Nautobot Network Provisioning

Thank you for your interest in contributing to the Nautobot Network Provisioning app! We welcome contributions of all kinds, including bug reports, feature requests, documentation improvements, and code changes.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub. Be sure to include:
- A clear and descriptive title
- Steps to reproduce the bug
- Expected vs. actual behavior
- Relevant screenshots or logs
- Your environment details (Nautobot version, Python version, OS)

### Feature Requests

If you have an idea for a new feature, please open an issue on GitHub to discuss it. We value community input and want to ensure that new features align with the project's goals.

### Developing and Submitting Code

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** to your local machine.
3.  **Create a new branch** for your changes: `git checkout -b feature/your-feature-name` or `bugfix/your-bug-name`.
4.  **Set up your development environment** (see the [README](README.md) for details).
5.  **Make your changes** and ensure they follow the project's coding standards.
6.  **Write tests** for your changes if applicable.
7.  **Run the tests** to ensure everything is working correctly.
8.  **Commit your changes** with clear and descriptive commit messages.
9.  **Push your changes** to your fork: `git push origin feature/your-feature-name`.
10. **Submit a pull request** from your fork to the main repository.

## Coding Standards

- We follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code.
- We use [Black](https://github.com/psf/black) for code formatting.
- We use [Flake8](https://gitlab.com/pycqa/flake8) for linting.
- We use [isort](https://github.com/timothycrosley/isort) for sorting imports.

## Development Environment

We recommend using the provided Docker-based development environment. See the [README](README.md) for instructions on how to set it up.

## Documentation

If you are contributing documentation, please ensure it is clear, concise, and follows the existing style.
